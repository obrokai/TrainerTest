# 遠距互動健身教練系統 — 系統設計

## 1. 使用者旅程
### 教練
1. 登入系統並選擇課程或建立新課程。
2. 開啟視訊後，透過手勢或控制面板切換「標註模式」。
3. 在畫面上比劃方向線、角度扇形或標記不正確的關節位置。
4. 系統自動分析學生的姿勢並提供建議，教練可確認後推送給學生。
5. 課程結束後，選擇是否保存錄影與標註。

### 學生
1. 進入課程，確認攝影機與麥克風權限。
2. 接收教練的視訊與空中標註，並在畫面上看到修正提示。
3. 若 AI 偵測到動作偏差，畫面會顯示提示文字或色彩提醒。
4. 課後可回放錄影、查看練習建議與教練備註。

## 2. 系統模組
| 模組 | 功能摘要 | 建議技術 |
| --- | --- | --- |
| 視訊串流 | 提供雙向視訊與音訊 | WebRTC、SFU (如 Janus、LiveKit) |
| 標註輸入 | 教練透過手勢或滑鼠繪製線條 | 手勢偵測（MediaPipe Hands）或 Canvas 繪圖工具 |
| 標註同步 | 將標註資料轉換成向量並同步給學生端 | WebSocket、DataChannel |
| 姿勢分析 | 偵測骨架關節、計算角度、比對範例 | MediaPipe Pose、TensorFlow.js、PyTorch |
| 回饋引擎 | 根據姿勢差異給出建議或警示 | 自訂規則引擎或 ML 模型 |
| 課程管理 | 課程排程、錄影保存、學員資料 | 後端 API (Node.js、Python/FastAPI)、資料庫 (PostgreSQL) |
| 雲端儲存 | 儲存錄影、標註、報表 | S3 兼容儲存（AWS S3、MinIO） |

## 3. 技術流程
1. **連線與驗證**：使用者登入後，透過 REST API 取得課程與權限資訊。
2. **視訊協商**：前端向訊號伺服器（Signaling Server）建立 WebRTC 連線並取得媒體串流。
3. **姿勢偵測**：
   - 前端執行輕量模型進行關節即時推論。
   - 若需要高精度，可將畫面或關節資料送往後端進行再分析。
4. **標註同步**：
   - 教練端的手勢或滑鼠輸入轉換成 `line`, `arrow`, `arc` 等元素資料。
   - 透過 WebRTC DataChannel 或 WebSocket 推送到學生端進行繪製。
5. **回饋顯示**：
   - 根據姿勢差異觸發提示（如「手肘角度需小於 90°」）。
   - 教練可選擇是否將 AI 建議呈現給學生。
6. **課後資料整理**：錄影與標註存檔，生成學員練習報表。

## 4. AI 與手勢偵測
- **手勢識別**：
  - 使用 MediaPipe Hands 或 Leap Motion 辨識教練手部動作。
  - 特定手勢（如捏合）代表開始繪圖，張開手代表停止。
- **姿勢評估**：
  - 收集教練示範動作的標準角度與節點。
  - AI 模型計算學生動作與標準差異，並轉換成語意化建議。
- **個人化訓練**：
  - 根據學員歷史數據調整評分門檻，提供更貼近能力的提示。

## 5. 安全與隱私
- 所有視訊與資料傳輸採用 TLS/DTLS 加密。
- 提供錄影使用者授權設定與到期自動刪除機制。
- 依地區法規確保個資保護（如 GDPR）。

## 6. MVP 開發順序建議
1. 建立基本雙向視訊與語音串流。
2. 加入 Canvas 繪圖與 WebSocket 同步線條功能。
3. 導入前端姿勢偵測並顯示基礎角度資訊。
4. 擴充 AI 回饋與課程管理模組。
5. 導入手勢控制、錄影回放與個人化推薦。

## 7. 後續擴充方向
- 多人課程與分組教學。
- 以 AR/VR 裝置提供更沉浸式的互動。
- 與穿戴裝置（心率、加速度）整合，提供完整訓練數據。
- AI 自動偵測動作錯誤並產生教練建議稿。
